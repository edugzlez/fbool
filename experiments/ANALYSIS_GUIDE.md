# Understanding CSV Outputs

This guide explains how to interpret the CSV files generated by the analysis pipeline.

## Bayes Optimal Analysis (`bayes_optimal_*.csv`)

### Columns

| Column | Description | Example |
|--------|-------------|---------|
| `input_file` | Path to input metrics parquet | `/data/npn_with_metrics.parquet` |
| `features` | Metric columns used for classification | entanglement, entropy, spectral_entropy |
| `mode` | Classification type | `binary` or `multiclass` |
| `target_rule` | Classification rule applied | `min_gates <= 9` |
| `count_column` | Weight column name (sample importance) | `count` |
| `correct_mass` | Number of functions correctly classified by majority rule | 1234567 |
| `max_accuracy_percent` | Maximum theoretical accuracy using these features (%) | 87.54 |

### Interpretation

The Bayes optimal analysis computes the **theoretical upper bound** on classification accuracy. It represents the best possible classifier given the features:

- **Binary mode**: Splits functions into two classes based on threshold (e.g., `min_gates <= 9`)
- **Multiclass mode**: Creates a separate class for each unique target value
- **Weighted analysis**: Each function counts by its `count` value (how many equivalent functions exist)

**Example:**
```csv
input_file,features,mode,target_rule,count_column,correct_mass,max_accuracy_percent
"npn_with_metrics.parquet","entanglement, entropy, spectral_entropy","binary","min_gates <= 9","count",1800000,75.00
```

Interpretation: Using 3 metrics, we can theoretically achieve 75% accuracy at best when classifying Boolean functions as simple (≤9 gates) or complex (>9 gates).

### Use Cases

- **Feature importance**: More predictive features should enable higher accuracy
- **Diminishing returns**: Compare accuracy with different feature subsets to identify which metrics matter
- **Benchmark**: Compare ML model performance against this upper bound

## Mutual Information Analysis (`mutual_info_*.csv`)

### Columns

| Column | Description | Example |
|--------|-------------|---------|
| `metric` | Name of the metric | entanglement |
| `I_Y_Xi` | Mutual information I(Y; metric) in bits | 0.523 |

### Interpretation

Mutual information I(Y; Xi) quantifies how much knowing metric Xi tells us about the target Y:

- **High I(Y; Xi)** (>0.1): Metric is very informative
- **Medium I(Y; Xi)** (0.01-0.1): Metric provides some information  
- **Low I(Y; Xi)** (<0.01): Metric has minimal predictive power  
- **Zero I(Y; Xi)**: Metric is completely independent of target

### Information-Theoretic Perspective

Information content is measured in bits: 1 bit = ability to answer one yes/no question.
- 1 bit MI ≈ metric can definitively answer "is target class A or B?"
- 0.1 bits MI ≈ metric helps narrow down target by ~7% optimally

**Example:**
```csv
metric,I_Y_Xi
entanglement,0.523
entropy,0.187
sensitivity,0.042
nolinearity,0.001
```

Ranking: entanglement >> entropy > sensitivity >> nolinearity

## Orthogonality Analysis (`orthogonality_*.csv`)

### Columns

| Column | Description | Example |
|--------|-------------|---------|
| `metric_a`, `metric_b` | Pair of metrics being compared | entanglement, entropy |
| `I_Y_a` | I(Y; metric_a) in bits | 0.523 |
| `I_Y_b` | I(Y; metric_b) in bits | 0.187 |
| `I_Y_ab` | I(Y; metric_a, metric_b) joint mutual information | 0.648 |
| `coinformation` | Interaction information C = I(Y;a) + I(Y;b) - I(Y;a,b) | 0.062 |
| `kappa` | Normalized coinformation κ = C / I(Y;a,b) | 0.0957 |
| `relationship` | Classification of relationship | REDUNDANCY |

### Relationships

**REDUNDANCY** (κ > 0, C > 0):
- Metrics provide overlapping information
- I(Y;a) + I(Y;b) > I(Y;a,b)
- Example: Two very similar circuit complexity metrics

**SYNERGY** (κ < 0, C < 0):  
- Metrics provide complementary information
- Neither alone fully explains the other's contribution
- Example: Different aspects of Boolean function structure

**ORTHOGONAL** (κ ≈ 0, C ≈ 0):
- Metrics are independent in their information
- I(Y;a) + I(Y;b) ≈ I(Y;a,b)
- Metrics capture different aspects of the target

### Mathematical Interpretation

**Coinformation C:**
```
C = I(Y;X₁) + I(Y;X₂) - I(Y;X₁,X₂)
```

- C > 0: Redundancy (duplicate information)
- C < 0: Synergy (complementary information)
- C = 0: Independence (orthogonal metrics)

**Kappa κ:**
```
κ = C / I(Y;X₁,X₂)
```

Normalized coinformation:
- κ ≈ 1: Nearly all joint information is redundant
- κ ≈ 0: Metrics are nearly orthogonal
- κ < 0: Strong synergy (complementary information)

### Example Analysis

```csv
metric_a,metric_b,I_Y_a,I_Y_b,I_Y_ab,coinformation,kappa,relationship
entanglement,entropy,0.523,0.187,0.648,0.062,0.0957,REDUNDANCY
entanglement,spectral_entropy,0.523,0.612,0.987,-0.076,-0.0770,SYNERGY
entropy,sensitivity,0.187,0.042,0.216,0.013,0.0602,REDUNDANCY
```

Interpretation:
- **entanglement + entropy**: Slight overlap (10% redundancy)
- **entanglement + spectral_entropy**: Complementary (7.7% synergy) → Use both!
- **entropy + sensitivity**: Weak overlap (6% redundancy)

## Reading the Results

### Step-by-Step Example

Given analysis files:
- `bayes_optimal_t9.csv` → Maximum achievable accuracy
- `mutual_info_t9.csv` → Individual metric importance
- `orthogonality_t9.csv` → Pairwise metric relationships

1. **Open bayes_optimal_t9.csv**: "Max accuracy is 78.5% with 5 metrics"
2. **Open mutual_info_t9.csv**: Rank metrics by I(Y; Xi) to identify top predictors
3. **Open orthogonality_t9.csv**: Find metric pairs with high synergy (κ < 0) for feature selection

### Model Development Guidelines

- **For feature selection**: Prefer orthogonal (κ ≈ 0) or synergistic (κ < 0) metric pairs
- **For explainability**: SYNERGY metrics reveal different aspects of the target
- **For efficiency**: REDUNDANCY metrics can often be dropped without much accuracy loss
- **Benchmark comparison**: ML model accuracy should approach max_accuracy_percent

## Advanced Usage

### Comparing Different Thresholds

```bash
bash scripts/quick_analysis.sh <metrics.parquet> ./results
```

Generates analyses with different thresholds:
- `bayes_optimal_t5.csv`: Simple functions (≤5 gates)
- `bayes_optimal_t9.csv`: Simple functions (≤9 gates)
- `bayes_optimal_multiclass.csv`: All gate counts

Compare max accuracy at different thresholds to find the most informative classification boundary.

### Subset Metrics

Run analysis with different metric combinations:

```bash
uv run cli.py experiments-bin bayes-optimal \
  --input metrics.parquet \
  --target min_gates \
  --binary --threshold 9 \
  --metrics metric1,metric2,metric3 \
  --output-csv results.csv
```

Generates `results.csv` with accuracy specific to those metrics.

Use this to understand feature importance and ensemble effects.
